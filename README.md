# Чёрный ящик

При реализации различных задач автоматизации, возникает необходимость по ведению архивов состояний. Помимо базовой комплектацией СУБД, часто возникает необходимость в независимом ведении архива - чёрный ящик. Данное решение, позволит разработчикам систем производить анализ по достоверным данным, исключающим умышленное изменение. Проект направлен на применение в одноплатных ПК, монтируемых в непосредственной близости от оборудования.

# Функциональные возможности

+ Архивирование данных
+ Поддержка различных протоколов
+ Формирование данных активности
+ Изолированность
+ Формирование отчётов
+ Импорт-экспорт конфигурации
+ Резервируемость линий связи
+ Ведение системных логов

Формирование данных активности - данные, генерируемые чёрным ящиком. Могут использоваться в технологическом оборудовании как признак готовности к работе. Таким образом соблюдается синхронность работы чёрного ящика и технологического оборудования.

# Содержимое проекта
+ cmd - точка входа (main.go)
+ docs - документация по проекту
+ pkg - пакеты проекта
  +  database - взаимодействие с БД
  +  libre - взаимодействие с libre
  +  loger - взаимодействие с логером
  +  modbusTCPmaster - взаимодействие с Modbus-TCP

# Запуск проекта
1. Перейти в директорию cmd: `cd ./cmd`
2. Выполнить построение исполняемого файла: `go build main.go`
3. Создать директорию, где будет расположен исполняемый файл
4. В созданную директорию, из раздела cmd, перенести:
   1. Файл .env
   2. Файл ConfigBB_inport.xlsx
   3. Файл main
5. В терминале перейти в созданную директорию и ввести: `./main`
